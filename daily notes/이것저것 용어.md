📌 CI/CD (Continuous Integration / Continuous Deployment)

- 코드를 수정하면 → 자동으로 검사하고 → 자동으로 배포까지 이어지게 만드는 개발 자동화 체계

1. CI (Continuous Integration) – 지속적 통합
    - 코드가 저장소에 들어올 때마다 “문제 없는지 자동 검사”
2. CD (Continuous Deployment / Delivery) – 지속적 배포
    - 검증된 코드가 자동으로 서버에 반영되도록 함


- Jenkins = “CI/CD 자동화 서버”

📌 JWT (JSON Web Token)

- JWT는 “로그인한 사용자가 누구인지 증명하기 위한 토큰”
 서버가 “이 사람은 로그인 성공했어”라는 사실을 토큰 하나에 담아서 프론트에 주고,
 프론트는 이후 요청마다 그 토큰을 같이 보내는 방식

- JWT = 서버가 발급한 ‘신분증 토큰’

📌 Docker란?

- Docker는 애플리케이션을 “실행 환경째로 포장해서 어디서든 똑같이 실행”하게 해주는 도구

- Docker = 실행 환경까지 포함한 프로그램 포장 기술

    1️⃣ Dockerfile (설계도)

    “이 컨테이너는 이렇게 만들어라”
    ```
    FROM node:18
    WORKDIR /app
    COPY . .
    RUN npm install
    CMD ["npm", "run", "dev"]
    ```

    - 어떤 OS

    - 어떤 언어 버전

    - 어떤 명령 실행

    2️⃣ Docker Image (완성된 실행 파일)

    Dockerfile을 build해서 만든 결과물

    변하지 않음 (불변)
    ```
    docker build -t my-app .
    ```


    👉 이미지 = 실행 준비 끝난 상태

    3️⃣ Docker Container (실행 중인 인스턴스)

    이미지를 run하면 컨테이너

    실제로 돌아가는 것
    ```
    docker run -p 3000:3000 my-app
    ```

    🧠 비유로 한 방에 정리
    개념	비유
    Dockerfile	요리 레시피
    Image	완성된 밀키트
    Container	실제 요리된 음식

📌 POC (Proof of Concept)란?

- POC는 “이게 진짜 가능한지 빠르게 증명해보는 실험용 구현”

- POC = 본격 개발 전에 ‘될지 안 될지’ 먼저 확인하는 미니 실험

📌 Grafana (그라파나)

- Grafana는 “서버·애플리케이션 상태를 실시간 그래프로 보여주는 모니터링 대시보드 도구”

- 숫자 로그를 그대로 보는 게 아니라  

    👉 한눈에 보이는 차트·그래프 화면으로 시각화해줌

HTTP (우리가 늘 쓰던 것)

클라이언트가 말 걸어야 서버가 대답

실시간에 부적합
